## CAPTCHA Class

The `captcha` class provides CAPTCHA (Completely Automated Public Turing test to tell Computers and Humans Apart) generation and verification functionality. It generates image-based CAPTCHAs with obfuscated characters to prevent automated bot submissions while allowing human users to verify their identity.

This class relies on the GD library for image manipulation and uses a daily rotating secret key to enhance security. CAPTCHAs are returned as base64-encoded PNG images embedded in data URIs for easy integration into web forms.

---

### Properties

| Name      | Value/Default | Description                                                                 |
|-----------|---------------|-----------------------------------------------------------------------------|
| `secret`  | `NULL`        | Daily rotating secret key used for CAPTCHA verification.                    |
| `length`  | `4`           | Number of characters in the CAPTCHA.                                        |
| `enabled` | `FALSE`       | Indicates whether CAPTCHA functionality is available (GD library required).|

---

### `__construct()`

#### Purpose
Initializes the CAPTCHA system. Checks for GD library availability, loads or generates a daily secret key, and enables CAPTCHA functionality if requirements are met.

#### Parameters
None.

#### Return Values
None (constructor).

#### Inner Mechanisms
1. Checks if the GD extension is loaded and supports PNG.
2. Retrieves the current secret key and its creation date from cache.
3. If the secret is outdated (older than today) or missing, generates a new one using `unique_id()` and updates the cache.
4. Sets `enabled` to `TRUE` only if all requirements are satisfied.

#### Usage Context
Automatically called when instantiating the `captcha` class. Ensures the system is ready for CAPTCHA generation and verification.

#### Example
```php
$captcha = new \cms\captcha();
if ($captcha->enabled) {
    // CAPTCHA is ready for use
}
```

### `create()`

#### Purpose
Generates a CAPTCHA image and verification code. The image contains randomly rotated characters with obfuscation to deter automated recognition.

#### Parameters
None.

#### Return Values
- **Array** or **Boolean**:
  - On success: Associative array with keys:
    - `code` (string): Verification code (hash of the CAPTCHA key and secret).
    - `image` (string): Base64-encoded PNG image as a data URI.
    - `width` (int): Image width in pixels.
    - `height` (int): Image height in pixels.
  - On failure: `FALSE` if CAPTCHA is disabled.

#### Inner Mechanisms
1. Generates a random lowercase alphanumeric key of length `$this->length`.
2. Computes a verification code using `hash32()` with the key and secret.
3. Creates a blank image with dimensions proportional to the key length.
4. For each character in the key:
   - Loads a pre-rendered character image from `CMS_IMAGES_PATH/captcha/`.
   - Rotates the character image randomly (±45 degrees).
   - Centers and copies the character into the main image.
5. Applies obfuscation by inverting pixel colors in the lower half of the image.
6. Converts the image to pure black and white to prevent shape recognition via intermediate shades.
7. Encodes the image as a base64 data URI and returns the result with metadata.

#### Usage Context
Used to generate CAPTCHAs for forms requiring human verification (e.g., login, registration, or comment forms).

#### Example
```php
$captcha = new \cms\captcha();
if ($captcha->enabled) {
    $result = $captcha->create();
    echo '<img src="' . $result['image'] . '" width="' . $result['width'] . '" height="' . $result['height'] . '" alt="CAPTCHA" />';
    echo '<input type="hidden" name="captcha_code" value="' . q($result['code']) . '" />';
    echo '<input type="text" name="captcha_key" placeholder="Enter CAPTCHA" />';
}
```

### `verify($key, $code)`

#### Purpose
Verifies whether a user-provided CAPTCHA key matches the expected verification code.

#### Parameters

| Name   | Type     | Description                                                                 |
|--------|----------|-----------------------------------------------------------------------------|
| `key`  | string   | User-provided CAPTCHA key (case-insensitive).                               |
| `code` | string   | Verification code generated by `create()` and submitted with the form.      |

#### Return Values
- **Boolean**:
  - `TRUE` if the key matches the code or CAPTCHA is disabled.
  - `FALSE` if the key does not match the code.

#### Inner Mechanisms
1. If CAPTCHA is disabled, returns `TRUE` (bypasses verification).
2. Computes a hash of the lowercase key concatenated with the secret.
3. Compares the computed hash with the provided code using `streq()`.

#### Usage Context
Used to validate user input after form submission. Typically called in form processing logic.

#### Example
```php
$captcha = new \cms\captcha();
if (isset($_POST['captcha_key']) && isset($_POST['captcha_code'])) {
    if ($captcha->verify($_POST['captcha_key'], $_POST['captcha_code'])) {
        echo "CAPTCHA verification successful!";
    } else {
        echo "Invalid CAPTCHA. Please try again.";
    }
}
```
